# Requirement ID: FR1 -- This is an example

* Description: \[The system shall allow users to log workouts without the application crashing.]
* Source Persona: \[Frequent Activity Tracker]
* Traceability: \[Derived from review group G1]
* Acceptance Criteria: \[Given the user is logging a workout When the logging process completes Then the application must not crash and the workout must be saved.]



# Requirement ID: FR1

* Description: \[The system shall allow users to start an anxiety or stress support session from the main interface in no more than 2 taps.]
* Source Persona: \[Anxious Student Seeking Fast Relief]
* Traceability: \[Derived from review group G1]
* Acceptance Criteria: \[Given the user is on the home screen When the user wants anxiety support Then the user must be able to open an anxiety or stress support session in no more than 2 taps.]



# Requirement ID: FR2

* Description: \[The system shall provide at least 3 guided coping options for anxiety or stress, such as breathing, grounding, or CBT-based reflection.]
* Source Persona: \[Anxious Student Seeking Fast Relief]
* Traceability: \[Derived from review group G1]
* Acceptance Criteria: \[Given the user starts an anxiety or stress support session When support options are displayed Then at least 3 distinct coping options must be available.]



# Requirement ID: FR3

* Description: \[The system shall allow users to record negative thoughts and receive a structured reframing prompt.]
* Source Persona: \[User Recovering from Depression and Negative Thought Cycles]
* Traceability: \[Derived from review group G2]
* Acceptance Criteria: \[Given the user enters a negative thought When the user submits the thought Then the system must return a structured reframing prompt or reflection exercise.]



# Requirement ID: FR4

* Description: \[The system shall provide supportive responses for users experiencing low mood or emotional distress without using dismissive language.]
* Source Persona: \[User Recovering from Depression and Negative Thought Cycles]
* Traceability: \[Derived from review group G2]
* Acceptance Criteria: \[Given the user describes low mood or emotional distress When the system generates a response Then the response must use supportive language and must not contain dismissive or mocking wording.]



# Requirement ID: FR5

* Description: \[The system shall allow users to begin a private conversational session at any time without requiring another human participant.]
* Source Persona: \[User Seeking a Safe Non-Judgmental Companion]
* Traceability: \[Derived from review group G3]
* Acceptance Criteria: \[Given the user opens the app When the user chooses to talk Then the system must allow the user to begin a conversational support session immediately without needing another user or therapist to join.]



# Requirement ID: FR6

* Description: \[The system shall maintain a supportive and non-judgmental tone throughout conversational interactions.]
* Source Persona: \[User Seeking a Safe Non-Judgmental Companion]
* Traceability: \[Derived from review group G3]
* Acceptance Criteria: \[Given the user is in a conversational session When the system replies Then the response must avoid insulting, blaming, or shaming language.]



# Requirement ID: FR7

* Description: \[The system shall clearly label whether each major feature is free, trial-based, or premium before the user attempts to access it.]
* Source Persona: \[Budget-Conscious User Evaluating Free vs Paid Value]
* Traceability: \[Derived from review group G4]
* Acceptance Criteria: \[Given the user is browsing app features When a feature is shown in the interface Then its access status must be displayed as free, trial-based, or premium before selection.]



# Requirement ID: FR8

* Description: \[The system shall display subscription pricing information before the user confirms a premium purchase.]
* Source Persona: \[Budget-Conscious User Evaluating Free vs Paid Value]
* Traceability: \[Derived from review group G4]
* Acceptance Criteria: \[Given the user selects a premium upgrade When the purchase screen appears Then the subscription price, billing period, and trial status must be displayed before confirmation.]



# Requirement ID: FR9

* Description: \[The system shall avoid repeating the same support message within 3 consecutive conversational turns unless the user requests repetition.]
* Source Persona: \[User Frustrated by Repetition, Weak Memory, or App Quality Issues]
* Traceability: \[Derived from review group G5]
* Acceptance Criteria: \[Given the user is in an active conversation When the system generates responses across 3 consecutive turns Then the same support message must not be repeated unless the user asks for it.]



# Requirement ID: FR10

* Description: \[The system shall preserve the context of the current conversation session so that user inputs are interpreted consistently within that session.]
* Source Persona: \[User Frustrated by Repetition, Weak Memory, or App Quality Issues]
* Traceability: \[Derived from review group G5]
* Acceptance Criteria: \[Given the user refers to information already shared in the current session When the system responds Then the response must remain consistent with the earlier user input from that session.]
