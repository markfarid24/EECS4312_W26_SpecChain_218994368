# Application: \[Wysa: Mental Wellbeing AI]



* Most important difference between pipelines: \[I think the manual pipeline made more focused review groups and personas in specific user situations, while the automated pipeline generated faster but broader outputs that sometimes grouped reviews too generally. The hybrid pipeline started from the automated results and improved them through manual refinement.]
* Most useful pipeline: \[I think the hybrid pipeline produced the most balanced outputs because it combined the speed of automation using human judgment. It preserved automation efficiency while improving clarity, traceability, and requirement quality.]
* Most surprising finding: \[I found many automated requirements that looked grammatically correct at first, but they were too broad. This shows that well-formed text is not always a good requirement.]
* Observed weakness in the automated pipeline: \[I noticed that the automated pipeline sometimes produced weird themes, unsupported persona details, and invalid review IDs.  Some automated requirements were also harder to test because they were too generic.]
